Cómo crear un checker para ejercicios de Minecraft

Builder web público para construir verificadores automáticos de ejercicios. Curso Algoritmika · Minecraft Global ESP / ENG.

🔗 Builder: teacher-ui.html
🛠️ Código fuente: github.com/FaintKom/mc-edu-checker-builder

📝 Cómo usar el Builder

  1. Abre el Builder en el navegador.
  2. Rellena Lesson info (título y descripción del ejercicio).
  3. Selecciona las comprobaciones que necesitas del catálogo (Mundo + Código).
  4. Ajusta los parámetros — por ejemplo: fn = agent.interact, min = 1.
  5. Pulsa Exportar y descarga el archivo ZIP.
  6. Guarda el ZIP con el nombre del ejercicio. Ejemplo: m2l4-task2-completando-primer-nivel.zip
  7. Sube el ZIP a la carpeta Drive del urok.
  8. Avisa al equipo de ensamblaje LMS que el checker está listo.
Convención de nombre: <módulo>-<lección>-task<N>-<descripción-corta>.zip
Ejemplos: m1l3-task1-laberinto-agente.zip · m2l1-task3-construir-granja.zip

🧰 Comprobaciones disponibles

El Builder ofrece más de 30 tipos de comprobaciones, divididas en dos categorías. Estas son las más usadas — el catálogo completo está dentro del Builder.

🌍 Mundo (.mcworld)

  • world.valid — el archivo es un mundo válido
  • world.name_contains — el nombre contiene texto X
  • world.gamemode_is — modo (creative / survival / adventure)
  • npc.min_count — al menos N NPCs
  • npc.all_have_dialogue — todos los NPCs tienen diálogo
  • entity.min_count_of — al menos N entidades de tipo X
  • sign.contains_text — algún letrero contiene texto X
+ otras 15 comprobaciones de mundo

💻 Código (.mkcd)

  • code.valid — el código compila
  • code.chat_command — comando de chat X existe
  • code.api_call_min — función X llamada ≥ N veces
  • code.has_loop — usa al menos un bucle
  • code.nested_loops — bucles anidados (profundidad ≥ N)
  • code.uses_block — agent.setItem con bloque específico
  • code.has_if — código contiene if / else
  • code.uses_identifier — usa identificador X
+ otras 5 comprobaciones de código

💡 Ejemplo común — validar agent.interact

Caso típico (ej. M2L4 Task 2)
«El ejercicio pide que el alumno use al menos una vez el bloque agent.interact en su código.»
Configuración en el Builder:
  • Añade la comprobación: code.api_call_min
  • Parámetro fn: agent.interact
  • Parámetro min: 1

Cuando el alumno suba su .mkcd, el checker analizará el código y mostrará verde si llama agent.interact al menos una vez, rojo si no.

❓ Preguntas frecuentes

¿Tengo que probar el checker antes de entregarlo?

Sí, recomendado. El Builder tiene una zona de pruebas integrada — arrastra un .mcworld o .mkcd de ejemplo y verifica que tus criterios funcionan como esperas.

¿Qué pasa si necesito un check que no está en el catálogo?

Avisa a @FaintKom (Mario). Los checks nuevos se añaden al código del Builder y quedan disponibles para todos en pocos minutos (GitHub Pages reconstruye automáticamente).

¿Funciona en cualquier navegador?

Chrome, Edge, Firefox y Safari modernos (últimos 2 años). No funciona en Internet Explorer.

¿Puedo guardar y reeditar un checker después de exportarlo?

Sí. Antes de exportar, copia el JSON de configuración del ejercicio (botón en el Builder). Lo puedes pegar después para seguir editando.

¿Funciona offline?

El Builder es estático pero carga librerías de CDN. Si necesitas usarlo offline, pide a Mario una copia descargable del HTML.
